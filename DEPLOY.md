# GitHub + Bothost.ru

## 1. Создать репозиторий на GitHub

### Через сайт GitHub
1. Войти в GitHub.
2. Нажать **New repository**.
3. Указать имя, например `kpop-chat-bot`.
4. Выбрать Public или Private.
5. Нажать **Create repository**.

### Загрузка через git
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/kpop-chat-bot.git
git push -u origin main
```

### Загрузка через веб-интерфейс
Можно просто открыть созданный репозиторий и загрузить все файлы через **Add file -> Upload files**.

## 2. Подготовка для Bothost

Перед деплоем:
- убедиться, что `requirements.txt` лежит в корне проекта;
- токен не хранить в коде;
- использовать путь БД `/app/data/bot.db` через переменную `DATABASE_PATH`.

## 3. Регистрация и запуск на Bothost

1. Зарегистрироваться на Bothost.ru.
2. Создать нового бота в панели.
3. Выбрать Python-проект.
4. Подключить GitHub-репозиторий или загрузить архив/файлы вручную.
5. Указать переменные окружения:
   - `BOT_TOKEN=...`
   - `DATABASE_PATH=/app/data/bot.db`
6. Команда запуска:
```bash
python main.py
```

## 4. SQLite и persistence

Для SQLite лучше хранить файл БД в `/app/data/bot.db`, потому что папка `data` предназначена для сохранения данных между обновлениями. Не хранить рабочую БД в корне репозитория.

## 5. Проверка после деплоя

1. Убедиться, что статус контейнера — `Running`.
2. Посмотреть логи старта.
3. Написать любое сообщение в групповой чат.
4. Проверить команду `инфо`.
5. Проверить покупку/баланс/работу.

## 6. Если репозиторий приватный

На Bothost можно подключить приватный репозиторий через токен с правами только на чтение.
